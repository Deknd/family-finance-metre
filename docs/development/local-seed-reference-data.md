# Локальный seed стартовой семьи

## Назначение

Этот документ описывает, как быстро завести стартовые данные в локальную БД для ручного тестирования сервера.

Seed нужен только для локальной разработки. Он не является Flyway-миграцией и не должен автоматически применяться в других окружениях.

## Что создает seed

Файл seed:

- `src/main/resources/db/manual/local_seed_reference_data.sql`

После запуска в БД будут созданы или обновлены:

- одна тестовая семья;
- два члена семьи;
- одно устройство;
- четыре зарплатных правила.

## Как запустить seed

Сначала нужно поднять локальную БД и запустить приложение или отдельно применить Flyway-миграции.

Пример команды через `psql`:

```powershell
psql "postgresql://family_finance_metre_db:family_finance_metre_db@192.168.1.70:5432/family_finance_metre_db" -f "src/main/resources/db/manual/local_seed_reference_data.sql"
```

Если у вас другой локальный Postgres, замените строку подключения на свою.

## Повторный запуск

Seed сделан идемпотентным:

- повторный запуск не должен создавать дубли;
- записи обновляются по фиксированным `id`.

Это удобно, если вы хотите вернуть локальную БД в предсказуемое демо-состояние.

## Как завести свою семью вместо demo-данных

Если нужна не demo-семья, а свои локальные данные:

1. Скопируйте `src/main/resources/db/manual/local_seed_reference_data.sql` в отдельный локальный SQL-файл.
2. Замените UUID, имена, Telegram-идентификаторы и устройство.
3. Убедитесь, что `telegram_chat_id` уникальны.
4. Для устройства заранее посчитайте новый `device_token_hash`.
5. Проверьте, что комбинации `(member_id, schedule_type, day_of_month)` не конфликтуют между собой.

## Важное замечание

Так как проверка `X-Device-Token` еще не реализована, raw token из этого документа нужен только как зафиксированное тестовое значение для будущих локальных сценариев. В БД уже хранится его совместимый hash.
