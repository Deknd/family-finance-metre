# Развитие проекта после MVP

## Назначение

Этот документ хранит идеи и направления развития после проверки базовой версии устройства.

## Возможные направления

- дополнительные экраны;
- прогноз до конца месяца;
- календарь обязательных платежей;
- категории расходов;
- цели накоплений;
- семейные финансовые цели;
- индекс финансового здоровья;
- уведомления об аномальных тратах.

## Принцип развития

Сначала нужно проверить, что пользователю удобны:

- короткий опрос;
- один основной экран;
- редкое обновление устройства;
- восприятие 3-4 ключевых показателей.

Только после этого стоит расширять сценарии.
