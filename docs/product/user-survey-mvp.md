# MVP-опрос пользователя для сбора финансовых данных

## Назначение

Этот документ описывает минимальный набор данных и сценарий опроса одного члена семьи для MVP.

## Минимальные данные

Нужно собрать только:

- сумму дохода по конкретной выплате, из-за которой был запущен опрос;
- актуальную оценку расходов конкретного члена семьи за весь текущий месяц на момент опроса;
- актуальную сумму платежей по кредитам за весь текущий месяц на момент опроса;
- сумму доступных накоплений этого человека на момент опроса.

## Базовые вопросы

1. `Сколько денег пришло в эту выплату?`
2. `Сколько всего у тебя выходит расходов за этот месяц?`
3. `Сколько всего у тебя выходит кредитных платежей за этот месяц?`
4. `Сколько денег сейчас есть в твоих накоплениях, которыми можно воспользоваться?`

## Как интерпретировать ответы

- один payroll-trigger = один intake payload;
- если в месяце было несколько выплат, `monthly_income` приходит отдельно по каждой выплате;
- сервер суммирует все значения `monthly_income` за месяц по одному человеку;
- `monthly_expenses` и `monthly_credit_payments` трактуются как актуальная оценка всего месяца на момент опроса;
- `liquid_savings` трактуется как текущее состояние накоплений на момент опроса.

## Принципы для LLM-опроса

- принимать приблизительные ответы;
- разрешать свободную форму;
- нормализовывать числа автоматически;
- задавать минимум уточнений;
- в конце показывать сводку и просить подтверждение.

## Что не спрашивать на MVP

- кредиты по отдельности;
- категории расходов;
- даты платежей;
- транзакции;
- инвестиции и прочие активы;
- детальную структуру бюджета.
