# Правила проекта

- На все публичные методы писать JavaDoc-комментарии с кратким описанием работы метода на русском языке.
- Для unit-тестов всех публичных методов делать проверки максимально читаемыми.
- Если для тестов нужна генерация данных, использовать библиотеку Instancio.
- В тестах использовать AssertJ и Mockito.
- Для тестов использовать понятные имена сценариев и добавлять краткое описание через `@DisplayName`.
- Максимально использовать Lombok, чтобы не писать лишний шаблонный код вручную.
- В Spring Boot использовать штатный функционал Spring Boot и Spring Framework, а не изобретать собственные механизмы без необходимости.
- Для валидации входящих данных использовать Jakarta Validation и Hibernate Validator.
